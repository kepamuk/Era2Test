# ERA2 — экран «Очередь генераций»

Тестовое задание: экран очереди с клиентским мок-движком, персистентностью и глобальным status bar.

## Запуск

```bash
npm install
npm run dev      # http://localhost:5173
npm run test     # vitest
npm run build
```

Страница очереди: **`/queue`**. Пункт есть в сайдбаре приложения.

## Роутинг

Используется собственный роутер проекта (`src/app/router/index.tsx`, `src/shared/routing`). Маршрут `/queue` рендерит `QueuePage` → виджет `GenerationQueue`.

`QueueProvider` и `GenerationStatusBar` смонтированы глобально в `App.tsx`, поэтому status bar доступен на всех страницах, кроме `/queue` (см. ниже).

## Архитектура (FSD)

| Слой | Путь |
|------|------|
| Сущность | `src/entities/generation-task` — типы и стартовый сид |
| Фича | `src/features/generation-queue` — reducer, engine, provider, UI, status bar |
| Виджет | `src/widgets/generation-queue` — композиция экрана |
| Страница | `src/pages/QueuePage.tsx` |

Публичный API слайсов — только через `index.ts` (`@/features/generation-queue`, `@/entities/generation-task`).

## Мок-движок

- `MAX_CONCURRENT = 2` — не больше двух задач в `running`, следующая `queued` берётся по FIFO (`createdAt`).
- Прогресс тикает каждые ~400–750 ms; шаг и задержка зависят от типа (`text`/`image` быстрее, `video`/`audio` медленнее).
- Сбой ~15% **на задачу** (один случайный порог прогресса 15–85%), сообщения из фиксированного списка.
- Таймеры снимаются при `cancel`, `fail`, `complete` и размонтировании провайдера.

Логика: `queueReducer.ts` + `queueEngine.ts`, единый стор через `QueueProvider` (`useReducer` + синхронный `stateRef` для движка).

## Персистентность (`localStorage`)

**Ключ:** `era2-generation-queue`

**Сохраняется:** список задач, фильтр по статусу, фильтр по типу, сортировка, строка поиска.

**Политика восстановления после перезагрузки:**

1. Если в `localStorage` есть валидные данные — они загружаются.
2. Все задачи в статусе `running` переводятся в `queued`, `progress` сбрасывается в `0` (имитация потери сессии; движок заново заполнит слоты).
3. Позиции в очереди и лимит параллелизма пересчитываются (`normalizeRestoredTasks` → `normalizeTasks`).

**Сброс к сиду:**

```js
localStorage.removeItem('era2-generation-queue')
```

После перезагрузки подтянется стартовый набор из `entities/generation-task/model/seed.ts` (10 задач).

**Эмуляция ошибки инициализации:** добавьте к URL `?queueInitError=1` — экран покажет состояние ошибки с кнопкой «Повторить».

## Глобальный status bar

Компонент `GenerationStatusBar` читает тот же контекст, что и страница очереди (`useQueue`). Счётчики и прогресс всегда синхронны.

**Скрытие на `/queue`:** на странице очереди индикатор не показывается — пользователь уже видит полный список; на остальных экранах бар ведёт себя как «менеджер загрузок» и ведёт на `/queue` по клику.

Состояния: нет активных → скрыт; 1 задача → компактная карточка; несколько → список + «Открыть очередь»; на desktop доступно свёрнутое состояние (пилюля).

## Шрифты

Основной — **Geist** (`@fontsource-variable/geist`), моно — **Geist Mono** для чисел и моделей. При отсутствии загрузки шрифтов браузер использует системный sans / monospace.
